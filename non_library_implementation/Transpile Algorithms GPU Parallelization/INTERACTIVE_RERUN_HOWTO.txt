INTERACTIVE RERUN MODE AND CUSTOM CLI GUIDE

This guide explains:
1. how to rerun any transpile algorithm scene directly from the CLI,
2. what scene labels each transpile script accepts,
3. which custom keys each scene takes,
4. what a custom command should look like,
5. how full-state / full-entanglement capture behaves by default.

The examples below use:
python "Transpile Algorithms Parallelization/<script>.py"


GENERAL USAGE

Default run:
python "Transpile Algorithms Parallelization/<script>.py"

Run one scene directly:
python "Transpile Algorithms Parallelization/<script>.py" A

Run one scene with custom parameters:
python "Transpile Algorithms Parallelization/<script>.py" A key1=value1 key2=value2

Equivalent alternate form:
python "Transpile Algorithms Parallelization/<script>.py" case=A key1=value1 key2=value2
python "Transpile Algorithms Parallelization/<script>.py" scenario=A key1=value1 key2=value2

Choose capture mode explicitly:
python "Transpile Algorithms Parallelization/<script>.py" --capture-mode full
python "Transpile Algorithms Parallelization/<script>.py" --capture-mode light

If you launch a transpile script interactively without capture flags, it now asks:
- Full: every-step state and entanglement capture, larger and slower
- Light: sampled checkpoints, faster and lighter
- Press Enter for Full

Important default:
- Full capture is now the default in the transpile suite.
- That means entanglement and state monitoring are on by default.
- Every-step capture is on by default.
- Full basis-state export is on by default.

Optional monitoring overrides:
- Disable all monitoring:
  python "Transpile Algorithms Parallelization/<script>.py" A --no-states --no-entanglement
- Increase capture limits for larger circuits:
  python "Transpile Algorithms Parallelization/<script>.py" A --state-qubits 12 --entanglement-qubits 12

BATCH RUNNER
Transpile Algorithms Parallelization/run_all_transpile_algorithms.py

Useful commands:
- Run one representative scene from every transpile script in full mode:
  python "Transpile Algorithms Parallelization/run_all_transpile_algorithms.py"
- Run all built-in scenes from every transpile script in full mode:
  python "Transpile Algorithms Parallelization/run_all_transpile_algorithms.py" --all-scenarios
- Filter to only one family:
  python "Transpile Algorithms Parallelization/run_all_transpile_algorithms.py" --only Grover
- Use a lighter sampled mode:
  python "Transpile Algorithms Parallelization/run_all_transpile_algorithms.py" --profile sampled


NOTES

- If you give custom parameters, always place the scene label first unless you use case=A or scenario=A.
- If a value contains spaces or shell-sensitive characters, wrap it in quotes.
- Some scenes are computationally heavy. Full capture mode can make them much slower and larger on disk.
- If a scene exceeds the full-state qubit limit, increase it with:
  --state-qubits <N> --entanglement-qubits <N>
